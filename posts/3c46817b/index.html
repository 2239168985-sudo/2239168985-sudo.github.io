⚠️ 重要口径声明：本文记录的是实验过程的原始台账，表中 mAP50 数字均为旧泄漏口径——后来证实当时使用的训练/测试划分存在泄漏，这些数字已被证伪、为虚高值，不能作为「某方案优于基线」的证据。真实口径下的结论请看本系列第三篇《关键发现》。保留本文是为了完整还原实验体系与决策过程。

2.1 注意力机制对比（B1–B11）

第一阶段对比了 11 个注意力 / 模块变体，包括 C2fPSA、BAM、SE、EMA、SPPFCSPC、C2fCBAM、C2fCA 等。代表性结果：

模型 改动 旧泄漏口径 mAP50（虚高，已被证伪）
baseline 原始 YOLOv8s 0.751
B1 C2fPSA 0.429（明显掉点，结构不友好）
B2 BAM 0.736（接近基线）

阶段性结论：局部加注意力大多低于或贴近基线；整体替换主干（如 B1 这一类改法）更伤精度。

2.2 数据增强与损失组合（B12–B20）

第二阶段尝试旋转、mixup、copy_paste、类别加权、过采样等组合。旧泄漏口径下表现最好的几组：

方案 旧泄漏口径 mAP50（虚高，已被证伪）
v4 旋转 0.772
v1 mixup 0.767
v2 cos 0.766
B19 过采样 0.761
B14 类别加权 0.759

⚠️ 这些「超越基线」的数字全部来自被污染的划分，不能直接当作方案有效的证据。后续用官方真实划分重验后，结论被彻底改写（见第三篇）。

2.3 网络结构改进（B21–B27）

进入第二周后，老师给出三点新指示：

  1. 针对草 / 水生植被识别设计专用结构；
  2. backbone 不是不能换；
  3. 把 YOLO 中可行的结构嫁接到 Swin / Mamba / Transformer 上。

按此方向完成 7 组结构改进实验：

模型 改动 旧泄漏口径 mAP50（虚高，已被证伪）
B21 RFAConv 草专属注意力（Neck P4） 0.728
B22 BiFormer 小目标路由注意力（Neck P3） 0.711
B23 ShuffleAttn 草专属注意力（Backbone 末） 0.714
B24 全 Swin 主干 backbone 整换 Swin-T 0.678
B25 ODMamba 主干 Mamba 从零训练（SSM 向量化提速 5×） 0.715
B26 Swin 并行支路 YOLO 嫁接 Swin（特征融合） 0.729
B27 SegNeXt 大核 水草大核注意力 0.741

本阶段的工程亮点

B25 的 ODMamba 主干最初用 Python for 循环逐 token 递推状态空间模型（SSM），GPU 利用率只有 41%，速度约 0.4 it/s。改为 GPU 并行前缀扫描（Hillis-Steele inclusive scan）后达到约 2.0 it/s，提速约 5 倍，100 个 epoch 的训练时间从约 17.6 小时降到约 3.3 小时，且数值与原实现等价（最大误差 1.9e-6）。这个优化的完整过程见本系列第四篇《工程实战》。

小结

三条路线、27 组实验在旧口径下看似「互有胜负」，但下一篇要讲的发现会让一切重新排序：评测口径不先搞对，所有对比都是空中楼阁。
